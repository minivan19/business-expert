# LLM Prompt约束规范

## 核心原则

### 1. 最小化原则
- 只包含必要信息，避免冗余
- 使用结构化数据，减少文本长度
- 明确分析要求，减少歧义

### 2. 标准化原则
- 使用统一的Prompt模板
- 保持一致的输出格式
- 遵循相同的约束规则

### 3. 效率原则
- 降低Token使用成本
- 减少API调用次数
- 提高响应速度

## 必须遵守的约束

### ❌ 禁止使用的格式

#### 1. 禁止Markdown格式标记
- **禁止使用** `**加粗**`、`*斜体*`、`~~删除线~~`
- **禁止使用** `## 标题`、`### 子标题`
- **禁止使用** `- 列表项`、`* 列表项`
- **禁止使用** `[链接](url)`、`![图片](url)`
- **禁止使用** `> 引用块`、```代码块```

#### 2. 禁止客套话和开场白
- **禁止使用** "好的"、"明白了"、"收到"
- **禁止使用** "作为您的"、"我已"、"这是"
- **禁止使用** "根据您提供的数据"、"经过分析"
- **禁止使用** 任何形式的问候语和结束语

#### 3. 禁止重复信息
- **禁止重复** Prompt中已有的信息
- **禁止重复** 数据摘要中的详细数据
- **禁止重复** 分析要求的文字描述

### ✅ 必须使用的格式

#### 1. 纯文本格式
- **使用** 普通段落文本
- **使用** 简单的编号：1）2）3）
- **使用** 自然的句子结构

#### 2. 直接输出
- **直接开始** 分析内容，不要任何前缀
- **直接回答** 分析要求，不要解释过程
- **直接提供** 结论和建议，不要铺垫

#### 3. 结构化输出
- **按部分** 组织分析内容
- **按要点** 列出关键发现
- **按优先级** 排序建议

## 数据准备规范

### 1. 数据清洗
- 移除特殊字符和乱码
- 统一数字格式（千位分隔符、小数位数）
- 标准化日期格式（YYYY-MM-DD）
- 处理空值和异常值

### 2. 数据摘要生成
```python
def generate_data_summary(data, summary_type):
    """生成数据摘要"""
    if summary_type == "subscription":
        return f"""
订阅统计：
年度订阅ARR：{data['2023_arr']}元，{data['2024_arr']}元，{data['2025_arr']}元
用户数量：{data['2023_users']}用户，{data['2024_users']}用户，{data['2025_users']}用户
单价：{data['unit_price']}元/用户/年
"""
    elif summary_type == "implementation":
        return f"""
实施费用统计：
2023年：{data['2023_total']}元
2024年：{data['2024_total']}元  
2025年：{data['2025_total']}元
年增长率：{data['growth_rate']}%
"""
```

### 3. Token数量控制
- 单个Prompt不超过2000 Token
- 数据摘要不超过800 Token
- 分析要求不超过300 Token
- 约束说明不超过200 Token

## 错误处理

### 1. 格式错误处理
```python
def validate_output_format(output):
    """验证输出格式"""
    errors = []
    
    # 检查是否包含禁止的Markdown
    if "**" in output or "##" in output:
        errors.append("包含禁止的Markdown格式")
    
    # 检查是否包含客套话
    forbidden_phrases = ["好的", "作为您的", "我已", "这是"]
    for phrase in forbidden_phrases:
        if phrase in output:
            errors.append(f"包含禁止的客套话: {phrase}")
    
    # 检查编号格式
    if re.search(r'^\d+\.\s', output, re.MULTILINE):
        errors.append("使用了错误的编号格式(应使用1)而不是1.)")
    
    return errors
```

### 2. 内容质量检查
- 分析要点是否完整
- 建议是否具体可行
- 风险识别是否全面
- 数据引用是否准确

### 3. 重试机制
- 格式错误：重新生成，最多3次
- 内容缺失：补充Prompt要求，重新生成
- API错误：等待后重试，最多3次

## 最佳实践

### 1. Prompt优化技巧
- 使用具体的数据，避免模糊描述
- 明确分析维度，减少歧义
- 提供参考标准，如行业平均、历史表现
- 设定输出长度预期，如"每个要点2-3句话"

### 2. 温度参数设置
- 分析类任务：temperature=0.2（低随机性）
- 创意类任务：temperature=0.7（中等随机性）
- 探索类任务：temperature=1.0（高随机性）

### 3. 批量处理优化
- 相似任务合并为单个Prompt
- 使用系统Prompt定义角色和约束
- 用户Prompt只包含数据和具体要求

### 4. 结果后处理
- 自动格式化编号（1. → 1））
- 移除多余空行和空格
- 验证关键数据准确性
- 添加时间戳和来源标记
